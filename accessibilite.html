Accessibilité

Manque de l'attribut title

Problème : Il y a dans le footer des liens qui dirige notamment vers les différents réseaux sociaux de l'IUT. Le problème est que les liens ne possèdent pas d'attribut title ce qui est très mauvais pour l'accessibilité. En effet, l'attribut titre permet d'afficher son contenu lorsque le lien est survolé par la souris. Il permet également à lire le lien par des machines à lire utilisées par les aveugles. Alors, sans cet attribut, certaines personnes ne peuvent pas comprendre à quoi sert le lien et où il dirige.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons les axes d'améliorations suivants :

  • Ajouter l'attribut "title" pour chaque lien (exemple : "Lien vers notre page Facebook").
  • Utiliser la classe "sr-only" qui est un nom de classe spécifiquement utilisé pour les lecteurs d'écrans.

Balise non appropriés

Problème : Dans la page "accueil.html", la balise span est utilisé pour un paragraphe. Il permet notamment de grouper certains éléments textuels afin de les mettre en forme avec le CSS. Or, span doit être utilisé lorsque aucun autre élément sémantique n'est approprié. Ici, la balise p serait plus appropriée. (Cela vaut également pour d'autres pages et notamment pour la page "debouchees.html" qui utilise "div", balise qui permet la division du contenu au lieu de la balise p). Les lecteurs d'écran utilisent les balises HTML pour se repérer dans la page, les utilisateurs de ses appareils pourraient ne pas comprendre l'usage de certaines balises

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons les axes d'améliorations suivants :

  • Remplacer la balise span par la balise p.
  • Remplacer la balise div par la balise p.

Mauvaise utilisation de la balise br

Problème : Dans la page "accueil.html", la balise br est utilisé deux fois de suite dans le but d'ajouter un espace entre les paragraphes, or, la balise br est utilisée lorsque les sauts de ligne ont une importance. L'utilisation de br représente ici une mauvaise pratique puisque la balise br sera indiquée pour les utilisateurs de lecteurs d'écran alors qu'elle n'apporte rien ce qui peut apporter une confusion chez l'utilisateur.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Utiliser la propriété "margin" en css à la place de la balise "br".

Niveau de titre sauté

Problème : Dans plusieurs pages du site, les niveaux de titre sont "sautés". Par exemple, dans la page "debouchee.html" on passe d'un titre de niveau 1 à un titre de niveau 7. Cela pose un problème d'accessibilité étant donné que les utilisateurs utilisant des lecteurs d'écran utilisent les niveaux de titres pour se repérer dans une page. L'absence d'un des niveaux peut donc amener de l'incompréhension chez l'utilisateur.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Respecter l'ordre des balises de titre : h1, h2, h3, h4, h5, h6.

Absence de balise de titre

Problème : Dans la page "accueil.html" par exemple, les noms de chaque compétence sont dans des balises p tout comme leur description, alors qu'ils devraient être dans une balise de titre étant donné que le nom de la compétence est plus important que la description. Les balises de titre sont particulièrement utiles aux personnes utilisant des lecteurs d'écran puisque celui-ci indique le niveau de titre.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Utiliser la balise de titre h3

Manque de l'attribut alt

Problème : Dans la page "accueil.html" et les autres pages, les images ne disposent pas de l'attribut alt. L'attribut alt permet d'afficher un texte ayant pour but de remplacer l'image. Cependant, étant donné qu'il n'y a pas d'attribut alt pour les images, si jamais une image ne charge pas l'utilisateur ne pourra pas savoir à quoi l'image correspond. L'attribut alt est également utile pour les utilisateurs de lecteur d'écran, sans cette information impossible de savoir l'information portée par l'image.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Ajouter un attribut alt pour chaque image (hormis les images décoratives).

Manque de la balise abbr

Statut : critique

Problème : Dans le header, il y a bien la présence de l'élément abbr concernant DUT MMI cependant il est absent pour celui du BUT MMI. L'élément abbr permet de fournir la signification d'une abréviation à travers l'attribut title. Il permet donc la bonne compréhension des abréviations dans un site. Alors, l'abréviation "BUT MMI" ne peut pas être comprise par l'utilisateur via le survol de la souris comme il peut le faire pour le DUT MMI.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Ajouter la balise abbr avec un attribut title correspondant pour chaque abréviation sur le site.

Utilisation de target="blank"

Problème : Dans le footer, le lien vers la page facebook utilise l'attribut target accompagné du mot clef "_blank" permettant d'ouvrir le lien dans un nouvel onglet. Cela peut être assez problématique notamment d'un point de vue sécurité étant donné que target="_blank" est utilisé sansrel="noreferrer" et "noopener" ce qui peut rendre le site vulnérable à des attaques via window.opener. De plus, il n'est pas précisé que le lien s'ouvre dans un nouvel onglet. Les personnes qui utilisent des lecteurs d'écran et les personnes souffrant de trouble de la vision peuvent être surpris que le lien s'ouvre dans un nouvel onglet (d'autant plus que cela s'applique uniquement pour la page facebook).

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons les axes d'améliorations suivants :

  • Ajouter une indication sur ce que se produira lorsque l'on clique sur le lien.
  • OU supprimer le target="_blank".

Image à la place de texte

Statut : critique

Problème : Dans la page "accueil.html", la section concernant les mentions légales est sous forme d'image alors qu'elle devrait être faite en utilisant les balises HTML comme "p", "h1" ou encore "a". Cela pose de nombreux problèmes avec notamment un problème de lisibilité (d'autant plus qu'il est appliqué une oppacité de 0.6 à l'image via le css) mais également un problème d'interaction puisqu'il est impossible de sélectionner le texte ou de cliquer sur un lien. Cela pose encore plus problème pour les utilisateurs utilisant un lecteur d'écran puisqu'ils ne pourront pas avoir accès au contenu textuel des mentions légales.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Remplacer l'image par du HTML en utilisant les balises appropriées.

Texte dans une autre langue que le français

Problème : Dans la page "accueil.html", dans la modale de gestion des cookies ou encore dans les erreurs du formulaire dans la page "contact.html", il y a des textes en langue étrangère (anglais et latin). Cela peut poser un problème de compréhension aux personnes ne parlant pas cette langue mais également pour les lecteurs d'écrans puisque, étant donné que la page HTML est difinie par lang="fr", la prononciation sera faussé et la compréhension encore plus compliquée.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Remplacer les textes écrits en anglais ou en latin par du Français.

Mauvaise lisibilité

Problème : Dans la liste des débouchés sur la page "debouche.html", il n'y a aucune marge entre les différentes boîtes contenant le nom du métier et la description. Cela pose un problème de lisibilité. De plus, lorsque l'on passe sur un terminal plus petit comme celui d'un iPhone, il est encore plus difficile de lire le texte.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons l'axe d'amélioration suivant :

  • Ajouter une marge entre chaque élément pour améliorer la lisibilité
  • Placer un élément par ligne sur des terminaux plus petits

Mauvais contraste de couleurs

Problème : Dans la page "contact.html", lorsque l'on sélectionne un article, le contraste entre le texte et le fond est très mauvais étant donné que le fond et le texte deviennent de la même couleur (blanc sur blanc). L'article et son prix deviennent donc illsibile : l'utilisateur n'a pas d'autres choix que de désélectionner l'article pour voir le texte.

Recommandation : Dans le cadre d'une démarche visant à améliorer l'accessibilité, nous vous recommandons les axes d'améliorations suivants :

  • Utiliser des couleurs ayant un contraste correct entre le fond et le texte.
  • Vous pouvez utiliser un Contrast Checker pour être certain que le contraste est correct. Exemple de contrast checker (ouverture dans un nouvel onglet)
