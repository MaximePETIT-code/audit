HTML/CSS

Pas de fichier "index.html"

Problème : La page d'accueil du site est nommé "accueil.html" cependant cela ne devrait pas être le cas. En effet, la page d'accueil d'un site web doit obligatoirement être nommée "index.html". Imaginons que l'adresse du site soit "https://www.butmmi.fr/". Lorsque l'utilisateur entre cette adresse, le navigateur cherche en réalité "https://www.butmmi.fr/index.html" et si jamais le fichier "index.html" n'existe pas, le navigateur indiquera que l'URL demandé n'a pas été trouvé. Ici pour accéder au site, on doit donc entrer l'adresse "https://www.butmmi.fr/accueil.html" ce qui représente un gros problème.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Renommer "accueil.html" par "index.html".

Absence du doctype

Problème : Dans la page "debouchees.html" on peut remarquer l'absence du préamble DOCTYPE html. Il s'agit d'un problème critique puisque le doctype est un composant essentiel d'un document HTML de qualité. En effet, les navigateurs se comportent de manière plus prévisible lorsqu'ils analysent des documents HTML commençant par une déclaration du doctype. Son but est d'empêcher un navigateur de passer en ce qu'on appelle le "quirks mode". (On peut également remarquer dans la page "contact.html" que le doctype est dans le iframe.)

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons les axes d'amélioration suivants :

  • Ajouter le doctype html en chaque début de page HTML.
  • S'assurer que le doctype soit bien au début de la page.

Les commentaires

Problème : Dans le code HTML pour la page "acceuil.html", des commentaires sont laissés. Bien que les commentaires ne soient pas pris en compte dans le contenu textuel du site internet, ils sont tout de même stockés et prennent donc de la place inutilement. De plus, ils sont accessibles depuis le code source de la page. Ici, cela s'avère être un problème critique puisque dans la page "accueil.html" un commentaire contient des données sensibles (nom d'utilisateur et mot de passe de la base de données).

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Retirer les commentaires lors de la mise en ligne du site.

Mauvaise identation

Problème : Dans la plupart des pages HTML du site, il y a des parties où l'identation est mauvaise. Par exemple, dans la page "accueil.html", les div contenus dans la balise main ne sont pas bien indentés. En effet, la balise main étant identé au niveau 2, les enfants de cette balise sont censés être indentés au niveau 3. Ce genre d'erreur peut causer certains conflits dans le code notamment lorsqu'on voudra gérer le CSS de la page.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Régler les problèmes d'indentations.

Mauvaise utilisation href

Problème : Dans la page "accueil.html", un hyperlien est défini dans une balise span en utilisant l'évènement onclick et localtion.href ce qui pose problème puisqu'on ne peut ouvrir la page dans un nouvel onglet par exemple. De plus, l'hyperlien est mauvais, il mène vers une page indisponible.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Utiliser l'élément d'ancre "a".

Attributs laissés vides

Problème : Dans le code HTML de chaque page, l'élement body possède les attributs class et style vides. Ils sont donc inutiles et prennent de la place dans le code pour aucune raison valable.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Ajouter un nom de classe et des propriétés dans style.

Utilisation de la méthode GET

Problème : Dans la page "debouchees.html", le formulaire utilise la méthode GET. La méthode get fait passer les données via l'URL. Cela pose un énorme problème de sécurité étant donné que les réponses au formulaire contiennent des données sensibles relatives à la carte de crédit.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons les axes d'améliorations suivants :

  • Utiliser la méthode POST.
  • Ajouter l'attribut action indiquant l'emplacement où les données doivent être envoyées.

Utilisation du type text

Problème : Dans la page "debouchees.html" les input concernant le numéro de la carte de crédit et le CVV sont des types texte. Malgré les patterns qui permettent que l'entré de caractère ne soit pas valide, il y a tout de même un problème. En effet, sur téléphone portable, le clavier s'adapte au type de champ (clavier numérique pour le type number et clavier alphabétique pour le type texte). Il est donc compliqué pour un utilisateur sur téléphone portable d'entrer son numéro de carte ainsi que le CVV.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons l'axe d'amélioration suivant :

  • Remplacer le type text par type number.

Nomenclature des fichiers

Problème : Les noms des fichiers (notamment les images) ou encore des id pour le formulaire de la page "contact.html" ne sont pas clairs et explicite ce qui amène à une perte de temps pour trouver à quoi renvoie le nom du fichier ou de l'id. (encore plus si plusieurs personnes travaillent sur le projet). De plus, les noms des images ont des espaces ce qui augmente le risque de faire une erreur lorsqu'on intègre l'image dans le code HTML.

Recommandation : Dans le cadre d'une démarche visant à améliorer le code HTML, nous vous recommandons les axes d'améliorations suivants :

  • Utiliser des noms d'images explicite.
  • Écrire le nom des images en Camel case ou remplacer les espaces par des "_".

Utilisation de !important

Problème : Dans la feuille de style du site, il est utilisé à plusieurs reprises la déclaration "!important". Il faut savoir qu'en css il existe des règles de cascade et d'héritage. La déclaration "!important" à la propriété d'annuler toute priorité dans le css ce qui remplace donc les règles de la cascade. Cependant, il est recommander d'utiliser !important uniquement en dernier recours et il peut totalement être éviter ici. (Le framework bootstrap utilise !important dans son css mais nous n'avons pas besoin de l'utiliser si la feuille de style est placée après celle de boostrap dans le head de la page HTML).

Recommandation : Dans le cadre d'une démarche visant à améliorer le code CSS, nous vous recommandons l'axe d'amélioration suivant :

  • Éviter l'utilisation de !important.
